本文档为 **RAG 电商导购 Agent 系统** 的部署和体验指南，帮助评委快速上手体验完整功能。

## 二、后端服务部署

### 2.1 环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| Python | 3.10+ | 推荐 3.10.x |
| pip | 21.0+ | Python 包管理工具 |
| 网络 | 可访问火山方舟 API | 需要配置 API Key |

### 2.2 部署步骤

```bash
# 1. 进入项目目录
cd /path/to/ragApp

# 2. 创建虚拟环境
python3.10 -m venv .venv
source .venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 复制环境变量配置
cp backend/.env.example backend/.env

# 5. 配置 API Key（关键步骤）
# 编辑 backend/.env，设置 LLM_API_KEY 或 ARK_API_KEY
```

### 2.3 环境变量配置详解

编辑 `backend/.env` 文件：

```bash
# ========== LLM 配置 ==========
# 火山方舟 API Key（必须配置）
LLM_API_KEY=your-ark-api-key-here
# 模型 ID（推荐使用豆包系列模型）
LLM_MODEL=doubao-seed-2-0-lite-260215
# 火山方舟 API 地址
LLM_BASE_URL=https://ark.cn-beijing.volces.com/api/v3

# ========== Embedding 配置（可选）==========
# 是否使用外部向量化模型
USE_EXTERNAL_EMBEDDING=false
# 使用内置 Chroma 嵌入模型时可忽略以下配置
EMBEDDING_MODEL=doubao-embedding-vision-251215
EMBEDDING_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
EMBEDDING_API_KEY=

# ========== 数据库配置 ==========
# Chroma 向量库路径（项目内已有预置数据）
CHROMA_PATH=./data/chroma
CHROMA_COLLECTION_NAME=agent_knowledge

# ========== 服务器配置 ==========
SERVER_HOST=0.0.0.0
SERVER_PORT=8000
CORS_ORIGINS=["*"]
```

### 2.4 数据库构建（可选）

项目已包含预置的 SQLite 商品库和 Chroma 向量库，通常无需手动构建。如需重新构建或更新数据，请按以下步骤操作：

#### 构建 SQLite 商品库

```bash
cd ecommerce_agent_dataset

# 增量构建（仅更新变更的商品）
python3 build_sqlite_db.py

# 完全重建（删除现有数据库后重新构建）
python3 build_sqlite_db.py --full-rebuild

# 指定数据路径和数据库路径
python3 build_sqlite_db.py --data-root . --db-path ecommerce.db
```

**构建说明**：
- SQLite 数据库包含商品、SKU、属性、FAQ、评价等完整信息
- 支持增量更新，仅处理源文件哈希变更的商品
- 构建完成后生成 `ecommerce.db` 文件

#### 构建 Chroma 向量库

```bash
cd ecommerce_agent_dataset

# 增量构建
python3 build_chroma_db.py

# 完全重建
python3 build_chroma_db.py --full-rebuild

# 指定数据路径和向量库路径
python3 build_chroma_db.py --data-root . --chroma-path .chroma
```

**构建说明**：
- Chroma 向量库包含商品营销文案、FAQ、评价的语义索引
- 使用配置的 embedding 模型进行向量化
- 构建完成后生成 `.chroma` 目录

**注意**：如使用外部 embedding 模型（如火山方舟），需先配置 `EMBEDDING_API_KEY` 环境变量。

#### 数据库文件位置

| 数据库 | 默认路径 | 说明 |
|--------|---------|------|
| SQLite 商品库 | `ecommerce_agent_dataset/ecommerce.db` | 商品、SKU、属性、FAQ、评价 |
| Chroma 向量库 | `ecommerce_agent_dataset/.chroma/` | 商品知识语义索引 |
| 语义表 | `ecommerce_agent_dataset/search_semantics/` | 商品概念、别名、fallback 关系 |

### 2.5 现有数据库下载（推荐）

为快速体验完整功能，推荐直接下载预置的数据库文件：

**下载方式**：

1. **飞书云文档下载**（推荐）：

## 三、Android 应用体验

### 3.1 开发环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| Android Studio | Hedgehog 2023.1.1+ | 推荐最新稳定版 |
| Gradle | 8.2+ | 项目已配置 |
| JDK | 21 | Android Studio 内置 |

### 3.2 运行步骤

1. **打开项目**：
   - 启动 Android Studio
   - 选择 `Open an existing project`
   - 选择 `android_app/` 目录

2. **等待 Gradle 同步**：
   - 首次打开会自动下载依赖
   - 确保网络畅通，同步完成后无错误

3. **配置后端地址**：
   - 运行应用后，点击右上角设置图标
   - 输入后端服务地址：
     - 模拟器访问宿主机：`http://10.0.2.2:8000`
     - 真机访问局域网：`http://192.168.x.x:8000`
   - 点击保存，配置立即生效

4. **运行应用**：
   - 选择模拟器或连接真机
   - 点击 Run 按钮（绿色三角）

### 3.3 Android 应用功能说明

| 功能区域 | 说明 |
|----------|------|
| 顶部状态栏 | 显示当前模型名称和向量模型连接状态 |
| 聊天输入框 | 输入导购问题，支持发送文本消息 |
| 消息区域 | 显示对话历史，支持 Markdown 渲染 |
| 商品卡片 | 助手回复下方展示推荐商品，支持点击 |
| 会话列表 | 底部导航切换，支持多会话管理 |
| 设置按钮 | 配置后端地址、管理模型列表 |

## 四、评委快速体验指南

### 4.1 推荐测试场景

#### 场景 1：探索浏览（随便看看）
```
随便看看有没有适合跑步的耳机
```
- 预期：识别为 `browsing` 模式，返回探索型回复
- 验证点：需求分析、商品推荐、卡片展示

#### 场景 2：明确购买
```
推荐一款防晒霜
```
- 预期：识别为 `purchase_ready` 模式，返回明确推荐
- 验证点：商品搜索、推荐理由、价格展示

#### 场景 3：多轮追问
```
推荐一款手机
再便宜点的呢？
```
- 预期：承接上一轮品类，返回更低价候选
- 验证点：上下文理解、价格筛选

#### 场景 4：品牌排除
```
除了耐克还有什么跑步鞋
```
- 预期：排除 Nike 品牌，推荐其他品牌
- 验证点：排除语义处理

#### 场景 5：商品对比
```
推荐两款防晒，对比一下价格和成分
```
- 预期：返回 Markdown 对比表，移动端转为卡片矩阵
- 验证点：对比意图识别、表格渲染

#### 场景 6：替代推荐（fallback）
```
肉松面包
```
- 预期：无直接命中时推荐替代品
- 验证点：fallback 机制、语义表匹配

### 4.2 关键功能验证清单

| 功能 | 验证方法 | 预期结果 |
|------|---------|---------|
| 流式响应 | 发送消息后观察 | 状态、分析、内容逐步显示 |
| 需求分析折叠 | 点击"已思考"区域 | 展开/折叠分析内容 |
| 商品卡片 | 点击卡片 | 跳转商品落地页 |
| 模型切换 | 点击顶部模型区域 | 切换服务端模型 |
| 会话管理 | 底部导航栏 | 新建、切换、删除会话 |
| 向量模型状态 | 观察顶部状态栏 | 显示连接状态和模型信息 |

## 六、常见问题与故障排除

### 5.1 后端启动失败

**问题**：`ModuleNotFoundError: No module named 'xxx'`
**解决**：确保已激活虚拟环境并安装依赖：
```bash
source .venv/bin/activate
pip install -r requirements.txt
```

**问题**：API Key 未配置导致认证失败
**解决**：编辑 `backend/.env`，配置 `LLM_API_KEY`

**问题**：端口被占用
**解决**：修改 `SERVER_PORT` 配置，或释放占用端口：
```bash
lsof -ti:8000 | xargs kill -9
```

### 5.2 Android 连接问题

**问题**：后端不可达
**解决**：
- 确保后端服务已启动
- 检查防火墙设置，允许 8000 端口
- 确保手机和电脑在同一局域网
- 正确配置后端地址（模拟器用 `10.0.2.2`）

**问题**：Gradle 同步失败
**解决**：
- 检查网络代理设置
- 确保 Android Studio 版本符合要求
- 手动下载依赖：点击 `File > Sync Project with Gradle Files`

### 5.3 功能异常

**问题**：商品搜索无结果
**解决**：检查 SQLite 数据库路径配置是否正确

**问题**：RAG 检索超时
**解决**：检查网络连接，或增加超时配置

**问题**：流式响应中断
**解决**：检查网络稳定性，后端会自动保存已生成内容

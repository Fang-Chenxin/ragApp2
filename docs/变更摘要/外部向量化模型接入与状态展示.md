# 外部向量化模型接入与状态展示 - 变更说明

## 变更时间
2026-06-08

## 变更概述

本次变更围绕 RAG 向量化能力完成后端固定配置化接入：后端支持使用火山方舟 OpenAI-compatible 多模态向量化接口构建和查询 Chroma 索引，Android 端只展示当前向量模型连接状态，不提供客户端切换入口。这样可以避免聊天模型选择和 embedding 模型混用，同时保证索引构建与线上检索使用同一套向量化配置。

---

## 后端变更

### 1. 配置管理 (`backend/config/settings.py`)

- **新增配置**:
  - `USE_EXTERNAL_EMBEDDING`: 启用外部向量化服务
  - `EMBEDDING_API_KEY` / `EMBEDDING_API_KEY_ENV`: 支持直接配置 API Key 或复用已有环境变量
  - `EMBEDDING_DIMENSIONS`: 配置外部 embedding 维度
- **兼容逻辑**:
  - 保留 `USE_DOUBAO_EMBEDDING` 作为旧开关兼容项
  - `resolved_embedding_api_key` 按 `EMBEDDING_API_KEY` → `EMBEDDING_API_KEY_ENV` → `LLM_API_KEY` 的优先级解析
- **路径固定**:
  - Chroma 默认路径固定到项目内 `ecommerce_agent_dataset/.chroma`
  - SQLite 默认路径固定到项目内 `ecommerce_agent_dataset/ecommerce.db`

### 2. RAG 服务 (`backend/service/rag_service.py`)

- **新增 `VolcengineMultimodalEmbeddingFunction`**:
  - 对接火山方舟 `/embeddings/multimodal` 接口
  - 实现 Chroma embedding function 所需的 `name()`、`get_config()`、`build_from_config()`、`default_space()` 等协议
  - 仅持久化非敏感配置，API Key 从运行时配置恢复
- **EmbeddingService 增强**:
  - 区分本地 embedding 与外部 embedding 状态
  - 新增 `get_status()`，返回 provider、model、base_url、dimensions、connected、message 等状态字段
  - 外部 embedding API Key 缺失时回退本地模型并记录原因
- **VectorStore 初始化增强**:
  - 外部 embedding 启用时自动确保 embedding service 已初始化
  - 捕获 Chroma embedding function 冲突，提示使用当前 `EMBEDDING_MODEL` 重建索引

### 3. 健康检查接口 (`backend/main.py`)

- `/health` 的 `services` 中新增 `embedding` 状态对象
- Android 端可通过健康检查获取后端固定向量模型连接状态

### 4. Chroma 构建脚本 (`ecommerce_agent_dataset/build_chroma_db.py`)

- 构建 Chroma 索引时复用后端同一套 embedding 配置
- 外部 embedding 启用时使用 `VolcengineMultimodalEmbeddingFunction`
- 构建日志输出模型、base_url、API 路径和维度，便于确认索引构建配置

## 用户使用说明

### 1. 使用本地 Chroma 默认 embedding

保持默认配置即可：

```bash
USE_EXTERNAL_EMBEDDING=false
```

### 2. 启用外部向量化模型

在 `backend/.env` 中配置：

```bash
USE_EXTERNAL_EMBEDDING=true
EMBEDDING_MODEL=doubao-embedding-vision-251215
EMBEDDING_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
EMBEDDING_DIMENSIONS=2048
EMBEDDING_API_KEY_ENV=ARK_API_KEY
```

或直接配置：

```bash
EMBEDDING_API_KEY=your_api_key
```

### 3. 重建 Chroma 索引

切换 embedding 模型或维度后，必须使用同一配置重建索引：

```bash
cd ecommerce_agent_dataset
python3 build_chroma_db.py --full-rebuild
```

### 4. Android 查看状态

启动聊天页或刷新模型列表时，Android 会自动请求 `/health`，并在聊天页顶部显示后端当前向量模型状态。

## 风险与注意事项

| 风险项 | 描述 | 缓解措施 |
|-------|------|----------|
| 索引与查询模型不一致 | 切换 embedding 模型后旧索引不可直接复用 | 切换模型后执行 `build_chroma_db.py --full-rebuild` |
| 外部 embedding 服务不可用 | API Key 缺失或接口无权限会影响外部向量化 | 健康检查展示状态，构建脚本启动时明确报错 |
| 向量维度变化 | 维度不同会造成 Chroma collection 不兼容 | 维度配置变更后重建 Chroma 索引 |

```bash
cd ecommerce_agent_dataset
python3 build_chroma_db.py --full-rebuild
```

4. 运行 RAG 查询或真实工具聊天流程，确认 Chroma 检索正常。

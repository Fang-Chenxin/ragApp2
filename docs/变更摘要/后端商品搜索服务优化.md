# 后端商品搜索服务优化 - 变更说明

## 变更时间
2026-05-29

## 变更概述
本次变更主要优化了后端的本地 SQLite 商品查询能力，包括：
- 增强基于自然语言的规则解析检索流程的降级与次相关推荐逻辑
- 规范并修正属性过滤器（`attr_filters`）的输入格式与处理
- 封装并强化 `SQLiteProductSearchService` 的可用性检测与模拟返回逻辑

---

## 后端变更

### 1. 模块 (`backend/service/product_search/query_tool.py`)
- **修改位置**: 整体重构降级逻辑与输入验证
- **新增/优化要点**:
  - 在自然语言检索无直接结果时，加入次相关商品推荐（secondary recommendation）机制：通过 `_build_secondary_recommendation_text()` 构造候选查询并重试搜索，成功时返回 `match_type=secondary_recommendation` 并带上 `primary_query_text` 与 `fallback_query_text`。
  - 优化了针对 `text` 查询无结果的提示与 `recommendation_hint`，便于前端展示友好提示。
  - 增加 `attr_filters` 标准化函数 `_normalize_attr_filters()`，支持两种输入格式：`[{"key":"颜色","value":"银色"}]` 或 `[["颜色","银色"]]`，并过滤无效条目。
  - 对工具规范 `get_tool_spec()` 做了声明增强，明确了参数类型、默认值和描述，方便 OpenAI Function Calling 调用。

### 2. 模块 (`backend/service/product_search/sqlite_search.py`)
- **修改位置**: 服务封装与日志/降级处理
- **新增/优化要点**:
  - 使用 `settings.sqlite_product_db_path` 配置数据库路径；若路径不存在，服务进入模拟数据模式并在返回中包含友好消息提示（`message` 字段）。
  - 封装两个入口：`search_by_rule_parsed_text()`（自然语言 + 规则解析）和 `search_products()`（结构化查询），两者在数据库不可用时返回模拟结果，保证上层服务稳定性。
  - 增强初始化日志与 `db_available` 属性判断，明确数据库可用性信息，便于运维定位。

## 核心问题修复清单
| 问题描述 | 修复状态 |
|---|---|
| 自然语言检索无结果时缺少可用的降级策略 | ✅ 已修复（新增次相关推荐重试逻辑） |

## 用户使用说明
1. 若使用工具调用 `query_products` 时传入 `text`，系统会先执行规则解析检索，若未命中会尝试构造次相关推荐查询并重试。
2. 传入 `attr_filters` 时可使用两种格式：`[{"key":"颜色","value":"银色"}]` 或 `[["颜色","银色"]]`。
3. 当后端未配置 SQLite 数据库（未设置或路径不存在）时，接口会返回模拟结果并在 `message` 字段给出提示，请通过 `sqlite_product_db_path` 环境变量或配置项指向有效数据库文件。

# 导购上下文排除与对比决策增强 - 变更说明

## 变更时间
2026-06-09

## 变更概述

本次变更围绕导购 Agent 的多轮上下文承接、否定排除、多商品对比决策和 Android 端对比结果展示展开。后端在 SearchPlan 中补齐追问、排除、价格偏好和对比意图等结构化字段，目标商品白名单过滤同步支持品牌、关键词和属性排除；最终回复在对比场景下强制生成结构化 Markdown 表格。Android 端不再直接用普通表格承载复杂对比，而是将 Markdown 对比表解析为“推荐排序 + 商品卡片矩阵”，更适合手机屏幕阅读。

---

## 后端变更

### 1. SearchPlan 上下文与对比字段增强

- **修改位置**: `backend/service/tool_chat/prompts.py`、`backend/service/tool_chat/product_selection.py`
- **新增字段**:
  - `is_followup`
  - `context_carryover`
  - `excluded_brands`
  - `excluded_terms`
  - `excluded_attributes`
  - `comparison_intent`
  - `comparison_dimensions`
  - `price_preference`
- **能力说明**:
  - 识别“再便宜点的呢”“上面那几款”“哪一个适合老人使用”等追问。
  - 识别“除了耐克”“不要含酒精的”等排除条件。
  - 识别“对比这两款”“哪一种更划算”“买哪款更值”等多商品取舍需求。
  - 对“性价比/更便宜/划算”类表达写入 `price_preference=cheaper`。

### 2. 目标商品召回与过滤增强

- **修改位置**: `backend/service/tool_chat/product_selection.py`、`backend/service/product_search/sqlite_search.py`
- **新增能力**:
  - 目标商品白名单过滤统一应用品牌、关键词、属性排除。
  - 匹配范围覆盖标题、品牌、类目、子类目、营销描述和全文检索文档。
  - `price_preference=cheaper` 时优先按价格排序，便于“再便宜点”“性价比高”类请求返回更合适候选。
  - RAG 候选、直查候选和工具候选合并后再次执行 SearchPlan 约束过滤，避免被排除商品回流。

### 3. 最终回复强约束与非白名单检测

- **修改位置**: `backend/service/tool_chat/stream_final.py`
- **新增能力**:
  - 直接回复不仅检查是否覆盖 primary 商品，还检测是否提到非白名单候选商品。
  - 如果直接回复提到非白名单商品，强制进入受约束最终回复。
  - 当 `comparison_intent=true` 且目标商品为 2-4 款时，即使直接回复覆盖商品，也强制进入结构化对比最终回复。
  - 最终 prompt 明确要求 GitHub Flavored Markdown 管道表格，表格前后留空行，第二行使用 `---` 分隔行，避免模型输出非标准表格。

### 4. Trace / Debug 验收字段补齐

- **修改位置**: `backend/service/tool_chat/trace.py`、`backend/service/tool_chat/stream_stages.py`
- **新增日志字段**:
  - SearchPlan 阶段展示追问、历史承接、排除品牌、排除关键词、排除属性、对比意图、对比维度、价格偏好。
  - 最终目标商品阶段展示本轮约束摘要。
  - 最终回复整理阶段展示是否覆盖目标商品、是否提到非白名单商品、是否因对比需求进入受约束重写。

## Android 端变更

### 1. Markdown 表格支持

- **修改位置**: `android_app/app/build.gradle.kts`、`ChatAdapter.kt`、`MainActivity.kt`
- **新增依赖**:
  - `io.noties.markwon:ext-tables:4.6.2`
- **说明**:
  - Markwon 初始化改为统一 builder，并启用 `TablePlugin`。
  - 保留对普通 Markdown 和简单表格的兼容能力。

### 2. 对比结果改为“推荐排序 + 商品卡片矩阵”

- **修改位置**: `android_app/app/src/main/java/com/example/agentchat/ChatAdapter.kt`、`item_chat_assistant.xml`
- **展示策略**:
  - 解析最终回复中的 Markdown 对比表。
  - 将表头中的商品列转置为商品卡片。
  - 将“推荐结论/结论”维度用于生成顶部推荐排序。
  - 每个商品卡片内部纵向展示价格、规格、卖点、适合人群、注意点、推荐结论等维度。
- **收益**:
  - 长文本自然换行，卡片高度随内容自适应。
  - 避免普通表格在手机屏幕上列宽平均、文字重叠、内容溢出的问题。
  - 多维度对比时不依赖横向滑动，更适合移动端导购阅读。

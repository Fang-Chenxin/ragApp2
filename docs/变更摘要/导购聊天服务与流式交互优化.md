## 2. 详细变更分析

### 2.1 后端服务层 (`backend/service/tool_chat_service.py`)

**核心功能增强:**

| 功能 | 说明 |
|------|------|
| **状态推送** | 新增 `_status_chunk()` 方法，实时推送任务状态（正在分析需求、正在查询商品、正在整理结果） |
| **异步工具调用** | 重构工具调用逻辑为异步执行，支持并发调用多个工具 |
| **选中商品提取** | 新增 `_extract_selected_products()` 方法，从工具结果中提取选中商品信息 |
| **选中商品回调** | 当有商品被选中时，推送 `selected_products` 类型消息 |
| **需求分析隔离** | 确保中间需求分析文本不直接流给用户，仅返回最终回复 |

**新增状态类型:**

| 状态 | 说明 | 额外信息 |
|------|------|---------|
| `analyzing_needs` | 正在分析需求 | - |
| `querying_products` | 正在查询商品 | tool_call_id, tool_name |
| `tool_error` | 工具调用失败 | error信息 |
| `tool_done` | 工具调用完成 | ok, total, elapsed |
| `organizing_results` | 正在整理结果 | - |

### 2.2 后端 API 层 (`backend/api/chat.py`)

**新增端点增强:**
- 支持流式工具聊天接口的状态消息传递
- 优化响应格式，支持 `selected_products` 类型消息

### 2.3 Android 客户端 (`MainActivity.kt`)

**新增功能:**
- 支持处理 `selected_products` 类型消息
- 移动端适配优化，支持状态消息展示

### 2.4 测试文件

**新增测试:** `test/test_tool_chat_service_streaming.py`
- 验证需求分析文本不会直接流给用户
- 验证非流式聊天也不会返回中间分析文本

## 4. 技术亮点

✨ **异步工具调用**: 支持并发执行多个工具调用，提升响应速度

✨ **状态实时推送**: 用户可以看到当前处理进度（分析需求→查询商品→整理结果）

✨ **选中商品回调**: 便于前端展示选中商品列表

✨ **需求分析隔离**: 确保中间分析过程不泄露给用户，只返回最终回复

✨ **降级逻辑优化**: 商品搜索失败时有优雅的降级处理

## 5. 用户使用说明

1. **正常使用**: 发送购物相关问题，系统自动分析需求、搜索商品并整理结果
2. **状态展示**: 聊天过程中会显示当前处理状态（正在分析需求、正在查询商品等）
3. **商品选中**: 当系统推荐商品时，会返回选中商品列表供前端展示

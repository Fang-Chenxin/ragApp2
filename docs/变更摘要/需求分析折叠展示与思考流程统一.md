# 需求分析折叠展示与思考流程统一 - 变更说明

## 变更时间
2026-05-29

## 变更概述
本次变更统一了后端"思考过程"与"需求分析"的数据流，将原有的 `thinking` 通道合并至 `analysis` 通道；Android 端移除了独立的思考气泡类型，改为在助手消息内嵌可折叠的需求分析区域，并新增 Markdown 渲染与复制回复能力。

---

## 后端变更

### 1. chat.py (`backend/api/chat.py`)

- **移除 `include_thinking` 参数**: `ChatRequest` 不再包含该字段，接口调用更简洁
- **移除 `thinking` chunk 类型处理**: 流式循环中不再处理 `type == "thinking"` 分支
- **统一 thinking 持久化**: 保存对话历史时，优先使用 `full_analysis` 作为 thinking 内容，降级使用 `analysis_summary`
- **新增保存日志**: 记录 `full_analysis_len`、`summary_len`、`thinking_to_save` 状态

### 2. tool_chat_service.py (`backend/service/tool_chat_service.py`)

- **移除 `include_thinking` 参数**: `chat_with_tools_stream()` 方法签名简化
- **需求分析兜底**: 分析结果为空时自动调用 `_build_need_analysis_summary()` 生成简化分析
- **日志级别调整**: 分析耗时、摘要日志从 `info` 降级为 `debug`，减少控制台噪音
- **tool 消息修复**: 构建最终消息列表时保留 `tool_call_id` 字段，兼容部分 LLM 提供方要求
- **移除 `chat_stream_with_thinking` 分支**: 最终 LLM 流式调用统一为 `chat_stream()`

---

## Android 端变更

### 1. ChatAdapter.kt (`android_app/.../ChatAdapter.kt`)

- **移除 `VIEW_TYPE_THINKING` 和 `ThinkingMessageViewHolder`**: 不再有独立思考气泡
- **新增 `Markwon` 支持**: 通过 `attachMarkwon()` 方法注入 Markdown 渲染器，助手消息与分析内容支持 Markdown 格式
- **助手消息重构**:
  - 新增可折叠分析区域（`analysisContainer`），展示"已思考"/"处理中"标题与耗时
  - 新增"复制回复"按钮，一键复制助手回复内容
  - 分析区域支持展开/折叠，状态通过 `analysisExpanded` 字段管理
- **文本选择支持**: 用户消息和助手消息均支持长按选择文本

### 2. MainActivity.kt (`android_app/.../MainActivity.kt`)

- **移除两个 Switch 控件**: 删除 `thinkingSwitch`（显示思考过程）和 `requestThinkingSwitch`（请求思考内容）
- **移除 `include_thinking` 字段**: `ChatRequest` 不再发送该参数
- **集成 Markwon**: 初始化时通过 `adapter.attachMarkwon(Markwon.create(this))` 注入
- **历史加载优化**: 使用 `notifyItemRangeRemoved/Inserted` 替代 `notifyDataSetChanged()`，配合 `post` 延迟滚动
- **对话框可选文本**: 服务器地址设置和清除历史对话框支持选择文本

## 用户使用说明

1. **需求分析查看**: 助手回复上方会展示"已思考"折叠区域，点击可展开查看分析内容
2. **复制回复**: 助手回复下方"复制回复"按钮，点击即可复制回复文本
3. **文本选择**: 长按任意消息可选择文本进行复制

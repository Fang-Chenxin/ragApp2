# 思考过程持久化与显示优化 - 变更说明

## 变更时间
2026-05-24

## 变更概述
本次变更彻底重构了大模型思考过程的处理逻辑，实现了思考内容永久持久化到历史记录文件，同时支持开关按钮自由控制显示/隐藏，解决了之前的多个严重问题。

---

## 后端变更

### 1. history_service.py - 扩展消息存储结构
- **修改位置**: `save_message()` 方法 (L220+)
- **新增特性**:
  - 新增可选参数 `thinking: Optional[str]`
  - 保存时自动对思考内容执行双重清洗，去除前后无意义空白换行
  - 仅当清洗后内容非空时，才写入 `thinking` 字段
- **向后兼容**: 完全兼容旧格式的历史记录，旧消息不会受任何影响

### 2. llm_service.py - 增强思考字段兼容性
- **修改位置**: `chat_stream_with_thinking()` 方法 (L145+)
- **新增兼容字段**:
  - `reasoning` (OpenAI 标准格式)
  - `reasoning_content` (豆包/火山引擎格式)
  - `thought` (其他主流模型格式)
- **兜底机制**: 增加基于 Pydantic 字典解析的兜底逻辑，即使属性访问失败也能提取到思考内容

### 3. chat.py - 核心流式逻辑重构
- **重大改动**: 无论 `include_thinking` 开关是 true 还是 false，始终在后台调用带思考的接口
  - 思考内容全程在后台累加、完整收集
  - 开关仅控制是否向前端实时推送思考显示片段
  - 彻底解决「关闭开关不保存思考内容」的问题
- **纯净化历史消息**: 构建给 RAG 服务的历史消息时，只保留 role 和 content 两个字段，完全剥离 thinking
- **修复累加 Bug**: 将 `full_thinking = thinking_content` 直接覆盖改为 `full_thinking += thinking_content` 累加模式，确保最终保存的思考过程 100% 完整

### 4. rag_service.py - LLM 调用层纯净化
- **修改位置**: 两个流式接口构建消息列表的部分
- **保障机制**: 传给大模型的所有历史对话，完全剔除 thinking 相关字段，确保思考内容绝对不会混入大模型的对话记忆

## Android 端变更 (MainActivity.kt)

### 1. 数据模型升级
- `ChatMessage`: 新增可选 `thinking` 字段，完整承载思考内容
- `HistoryMessage`: 新增可选 `thinking` 字段，与后端 JSON 结构对齐

### 2. 流式显示逻辑重构
- **不再提前删除思考气泡**: 思考消息气泡一旦显示就完整保留，不会在正式内容开始后自动消失
- **索引安全管理**: 思考消息正确插入在助手主消息之前，保证界面布局顺序绝对正确，彻底解决排版混乱
- **全程累加收集**: 无论开关状态，`fullThinking` 始终收集所有思考片段，确保助手主消息最终带上完整思考内容

### 3. 历史记录加载优化
- 切换思考显示开关时，强制从服务器重新拉取完整历史记录
- 完全清空本地消息列表，根据当前开关状态 100% 重新构建所有消息气泡
- 一次切换立即生效，无需用户重复点击两次开关

### 4. 历史请求防错机制
- 发送给后端的 `ChatRequest` 消息列表：
  - 过滤掉本地临时的 `thinking` 显示消息
  - 强制将所有消息的 thinking 属性置空
  - 历史消息结构 100% 纯净化，彻底杜绝 400 错误
- **完全隔离**: thinking 属性只作为本地显示和传递给后端存储使用，绝不参与后续对话轮次的历史记忆

## 用户使用说明

1. **显示思考**: 打开界面右上角的思考显示开关，即可在对话过程中实时看到完整的思考过程流式输出
2. **隐藏思考**: 关闭思考显示开关，界面不显示思考内容，但思考过程仍然 100% 完整保存在后台历史记录文件中
3. **查看历史**: 任何时候切换开关，历史记录都将立即按当前设置重新渲染，之前关闭开关时对话保存的思考过程现在可以立刻显示出来
4. **向后兼容**: 升级不影响你之前的所有历史对话记录，完全兼容旧格式

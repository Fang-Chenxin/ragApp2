# 导购流程并行化与商品卡片联动优化 - 变更说明

## 变更时间
2026-06-09

## 变更概述

本次变更围绕导购聊天链路的候选商品收敛、商品详情页展示、Android 端结构化数据兼容和测试口径同步展开。核心目标是让后端能够更稳定地把 RAG 候选、商品直查结果和工具查询结果合并成可核验的目标商品白名单，同时让前端对 `selected_products`、`sku_id`、空字段和商品页跳转保持兼容。

---

## 后端变更

### 1. 聊天流式接口 (`backend/api/chat.py`)

- **修改位置**: 流式 SSE 事件分发逻辑
- **新增特性**:
  - `status`、`analysis`、`content`、`thinking` 等事件字段统一补齐为空字符串，避免前端解析时遇到缺字段。
  - `selected_products` / `selected_product_ids` 在各阶段中统一输出，保证事件结构稳定。
- **收益**:
  - Android 端可以直接按固定字段消费流式响应，不再依赖字段是否存在。

### 2. 商品搜索 API (`backend/api/sqlite_product_search.py`)

- **修改位置**: 商品详情接口、落地页接口、图片路径解析与 HTML 生成
- **新增特性**:
  - 商品详情改为直接读取 SQLite 中的商品、SKU、属性、FAQ 和评价，返回更完整的结构化数据。
  - 商品落地页支持 `sku_id` 参数，可按指定规格默认展开页面。
  - 新增图片路径解析兜底逻辑，支持按文件名在 dataset 的 `images` 目录中回查历史路径。
  - 落地页 HTML 改为更完整的商品详情页样式，包含 SKU 卡片、属性表、FAQ 和评价区块。
- **收益**:
  - 商品页可以展示更完整的结构化信息，也能兼容历史图片路径变动。

### 3. 商品查询工具层 (`backend/service/product_search/query_tool.py`)

- **修改位置**: Tool Spec 和 `run_tool()`
- **新增特性**:
  - `category` / `sub_category` 的 JSON Schema 依据数据库真实类目动态生成枚举。
  - 新增类目合法性校验，非法类目会直接返回 `validation_error`。
  - 当 `text` 与结构化约束同时存在时，优先保留约束，避免自然语言泛词跨类目召回。
- **收益**:
  - LLM 更难输出伪类目，工具查询结果更贴近数据库真实枚举。

### 4. 商品检索服务 (`backend/service/product_search/sqlite_search.py`)

- **修改位置**: 商品库初始化、类目树读取、商品精确回查
- **新增特性**:
  - 增加真实类目树缓存与校验接口 `validate_category_filters()`。
  - `get_products_by_ids()` 改为显式创建和关闭 SQLite 连接。
  - 新增默认 SKU 回查能力，供落地页和推荐卡片复用。
- **收益**:
  - 搜索服务更适合作为目标商品白名单的校验层，避免脏类目和无效 ID 流入最终推荐。

### 5. 导购目标商品选择 (`backend/service/tool_chat/product_selection.py`)

- **修改位置**: SearchPlan 标准化、候选商品合并、约束过滤和最终推荐构造
- **新增特性**:
  - 使用语义表修正 `target_category`、`target_sub_category` 和 fallback 约束。
  - 将 RAG 候选、工具结果和后台直查结果统一合并成目标商品白名单。
  - 对工具参数做 SearchPlan 补全，并记录原始参数、执行参数和补全原因。
  - 最终推荐会根据用户显式品类约束、语义词和 direct/fallback 规则过滤候选。
- **收益**:
  - 推荐链路更确定，能减少“查到但不该推荐”的商品混入最终输出。

### 6. 工具循环与最终回复 (`backend/service/tool_chat/stream_tool_loop.py`, `backend/service/tool_chat/stream_final.py`)

- **修改位置**: 工具循环提前退出、最终内容生成顺序、回复覆盖判定
- **新增特性**:
  - 当 RAG/直查已经得到足够候选时，可以跳过后续工具循环。
  - 工具结果为空但已有可核验目标商品时，提前进入最终推荐阶段。
  - 最终回复会先发一段可见正文，再输出商品卡片，减少前端长时间空白。
  - 新增 direct reply 覆盖检查：若 LLM 直接回复已经提到了 primary 商品，就避免重复受约束重写。
- **收益**:
  - 流式体验更稳定，推荐结果更快进入可见状态，且减少不必要的重复生成。

### 7. 调试追踪 (`backend/service/tool_chat/trace.py`)

- **修改位置**: 工具计划、参数补全、目标商品阶段日志
- **新增特性**:
  - 记录原始参数与补全后参数。
  - 记录 SearchPlan 补全原因与 tool call 的执行差异。
  - `selected_products` 阶段额外显示 RAG 候选。
- **收益**:
  - 方便快速定位“模型怎么想的”和“后端怎么补的”之间的差异。

## Android 端变更

### 1. 消息结构兼容 (`android_app/app/src/main/java/com/example/agentchat/MainActivity.kt`)

- **修改位置**: 流式响应解析、历史消息回填、商品结构体
- **新增特性**:
  - `SelectedProduct` 新增 `skuId` 字段。
  - 对 `content`、`analysis`、`status`、`selectedProducts` 做空值防护，兼容后端缺字段或 Gson 反序列化的 `null`。
  - `getBackendUrlWithPath()` 改为自动处理斜杠，避免重复拼接路径。
- **收益**:
  - 流式解析更稳，不容易因为空字段导致 UI 更新异常。

### 2. 商品卡片交互 (`android_app/app/src/main/java/com/example/agentchat/ChatAdapter.kt`)

- **修改位置**: 图片加载、商品页打开、后台地址拼接
- **新增特性**:
  - 商品图片 URL 统一走后端地址解析。
  - 本地 `imagePath` 支持按 dataset 图片接口转换成可访问 URL。
  - 打开商品页增加异常捕获，避免找不到外部应用时直接崩溃。
- **收益**:
  - 商品卡片在后端地址变化、图片路径变动或设备缺少浏览器时都更稳。

### 3. 商品卡片布局 (`android_app/app/src/main/res/layout/item_product_card.xml`)

- **修改位置**: 外边距属性
- **新增特性**:
  - 将错误的 `marginEnd` 改为 `layout_marginEnd`。
- **收益**:
  - 修复布局属性不生效的问题。

## 用户使用说明

1. 普通用户的导购流程无需改变，聊天入口和商品卡片仍可正常使用。
2. Android 端现在可以更稳地展示流式回复、商品卡片和商品页跳转。
3. 后端开发调试时，可以通过 trace 日志查看 SearchPlan 补全原因、工具参数变化和 RAG 候选。
4. 测试时可优先运行 `test/test_history_and_stream_safety.py` 和 `test/test_conversation_scenarios.py`，快速确认流式与推荐链路是否正常。

## 风险与后续计划

| 风险项 | 描述 | 缓解措施 |
| --- | --- | --- |
| 商品落地页 HTML 变复杂 | 页面结构更完整，但维护成本更高 | 后续可考虑拆出模板渲染层 |
| SearchPlan 规则继续扩展 | 语义表和约束会持续增多 | 后续可将规则进一步配置化 |
| Android 端仍依赖后端字段约定 | 新字段增多后可能需要继续同步客户端 | 继续以结构化字段而非文本解析为准 |

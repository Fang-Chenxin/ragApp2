# 后端稳定性与测试同步优化 - 变更说明

## 变更时间
2026-06-07

## 变更概述

本次变更围绕后端导购聊天链路的稳定性、并发安全和测试口径同步展开。核心目标是先处理代码审查中风险较高、改动范围可控的问题，再为后续拆分 `chat_with_tools_stream` 做低风险铺垫。

---

## 后端变更

### 1. 对话历史服务 (`backend/service/history_service.py`)

- **修改位置**: `HistoryService` 文件读写与 `save_message()` 写入流程
- **新增特性**:
  - 使用 `fcntl.flock` 对会话文件和元数据文件加文件锁
  - 新增原子 JSON 写入流程，先写临时文件再 `os.replace`
  - `save_message()` 在同一会话文件锁内完成历史追加和会话元数据计数更新
- **修复问题**:
  - 修复并发请求下“读取 → 追加 → 全量写回”导致的消息丢失风险
  - 降低异常中断时写出半截 JSON 的概率
- **向后兼容**: 是，仍使用原有 JSON 文件结构

### 2. 聊天 API (`backend/api/chat.py`)

- **修改位置**: `generate_stream()` 流式响应生成器、模型发现流程
- **新增特性**:
  - 新增 `save_history_once()`，统一历史保存逻辑
  - 在流式生成器 `finally` 中保存已生成的 assistant 内容
  - 模型发现增加 `asyncio.Lock` 与缓存 double-check
- **修复问题**:
  - 修复客户端断连或流式过程异常时，用户已看到的回复没有保存到历史的问题
  - 避免并发刷新模型列表时重复发起模型发现请求
- **向后兼容**: 是，SSE 数据结构保持兼容

### 3. 工具聊天服务 (`backend/service/tool_chat_service.py`)

- **修改位置**: `chat_with_tools()`、`chat_with_tools_stream()` 及工具规划消息构造
- **新增特性**:
  - 非流式工具调用改为 `await asyncio.to_thread(run_tool, ...)`
  - 新增 `_StreamTaskGroup`，统一追踪流式期间创建的后台任务并在退出时取消未完成任务
  - 新增 `_build_tool_planning_messages()`，流式与非流式共用工具规划消息构造逻辑
- **修复问题**:
  - 降低非流式工具查询阻塞事件循环的风险
  - 修复流式连接提前退出时后台任务继续运行的资源浪费问题
- **向后兼容**: 是，流式事件协议保持不变

### 4. SQLite 商品搜索服务 (`backend/service/sqlite_product_search_service.py`)

- **修改位置**: `get_products_by_ids()`
- **新增特性**:
  - SQLite 连接改为显式创建、查询、关闭
- **修复问题**:
  - 修复依赖 `with sqlite3.connect(...)` 造成连接生命周期不够清晰的问题
- **向后兼容**: 是

### 5. 查询引擎 (`backend/service/query_engine.py`)

- **修改位置**: `load_ontology()`、`build_reverse_index()`
- **新增特性**:
  - `load_ontology()` 增加 `lru_cache`
  - 反向索引构建增加轻量缓存
- **优化效果**:
  - 减少每次商品查询时重复读取 ontology 文件和重复构建反向索引的开销
- **向后兼容**: 是

## 测试变更

### 1. 新增稳定性测试 (`test/test_history_and_stream_safety.py`)

- **新增覆盖**:
  - 并发 `save_message()` 不丢消息
  - 流式生成中断后保存已生成回复
  - `_StreamTaskGroup` 可取消未完成任务
  - `_build_tool_planning_messages()` 保持 system/history/user 顺序

### 2. RAG 测试同步 (`test/test_rag_vector_query.py`)

- **修改内容**:
  - 按正式版行为校验结构化 `metadata.product_id`
  - 不再依赖旧版格式化文本中的 `商品ID: ...`
- **说明**:
  - 当前 RAG 向量模型召回质量仍偏弱，后续计划升级为更先进的 embedding / rerank 模型

### 3. 真实工具聊天流程测试 (`test/test_real_tool_chat_flow.py`)

- **修改内容**:
  - 不再强制要求返回 `rag_sources`
  - 改为校验 `vector_search` 和 `rag_rerank` 调试明细
  - 最终商品 ID 校验以 `selected_products` 结构化事件为准
- **原因**:
  - 正式流程允许 RAG 核验丢弃不相关片段，不能把 `rag_sources` 作为必然产物

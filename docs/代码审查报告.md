## 二、🔴 Critical 问题

### C1. `chat_with_tools_stream` 方法长达 800+ 行（God Method）

**文件**: `tool_chat_service.py` L1230-L2043

**问题**: 单个方法承载了并行任务管理、RAG 检索、工具循环、回复生成、SSE 事件分发等所有职责，包含 6 个闭包函数，极难维护和测试。

**优化方案**: 拆分为独立的阶段方法：

```python
class ToolChatService:
    async def chat_with_tools_stream(self, ...):
        ctx = StreamContext(user_query, conversation_history, model, model_config)
        
        # 阶段 1：并行启动
        yield self._status_chunk("正在分析需求", "need_analysis")
        await ctx.run_parallel_tasks()
        async for event in ctx.drain_analysis_events():
            yield event
        
        # 阶段 2：RAG 检索 + 重排序
        yield self._status_chunk("正在检索知识库", "retrieving_knowledge")
        await ctx.run_rag_pipeline()
        yield ctx.vector_trace_chunk
        yield ctx.rerank_trace_chunk
        
        # 阶段 3：工具调用循环
        async for event in ctx.run_tool_loop():
            yield event
        
        # 阶段 4：最终回复生成
        async for event in ctx.generate_final_reply():
            yield event
```

### C2. `history_service` 并发写入不安全（Lost Updates）

**文件**: `history_service.py` L220-L240

**问题**: `save_message` 采用 "读取 → 追加 → 全量写回" 模式，无任何锁机制。并发请求会导致消息丢失或 JSON 损坏。

```python
# 当前代码（危险）
history = self.load_history(user_id, conv_id)  # 读
history.append(message)                         # 改
with open(file_path, 'w') as f:                 # 写
    json.dump(history, f)
```

**优化方案 A**（最小改动 - 文件锁）：
```python
import filelock

def save_message(self, user_id, conv_id, role, content, thinking=None):
    lock_path = self._get_conv_file(user_id, conv_id) + ".lock"
    with filelock.FileLock(lock_path, timeout=5):
        history = self.load_history(user_id, conv_id)
        history.append(message)
        self._atomic_write(file_path, history)  # 先写 .tmp 再 rename
```

**优化方案 B**（推荐 - 迁移到 SQLite）：
```python
# 利用项目已有的 SQLite 基础设施
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT NOT NULL,
    conv_id TEXT NOT NULL,
    role TEXT NOT NULL,
    content TEXT NOT NULL,
    thinking TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### H2. 流式响应中断导致对话历史丢失

**文件**: `api/chat.py` L430-L460

**问题**: `save_message` 仅在收到 `done` 事件时执行。客户端断连时 `yield` 抛异常，循环中断，已生成的回复不会保存。

**修复**：
```python
async def generate_stream():
    full_reply = ""
    try:
        async for chunk in tool_chat_service.chat_with_tools_stream(...):
            # ... 现有逻辑 ...
            if chunk_type == "content":
                full_reply += content
    except (asyncio.CancelledError, Exception):
        pass  # 客户端断连
    finally:
        # 无论何种退出，都保存已生成的内容
        if full_reply.strip():
            history_service.save_message(user_id, conv_id, "user", request.user_query)
            history_service.save_message(user_id, conv_id, "assistant", full_reply)
```

### H3. 并行任务缺乏取消机制（资源泄漏）

**文件**: `tool_chat_service.py` L1506-L1510

**问题**: `asyncio.create_task` 创建的任务在流式连接断开时不会被取消，浪费服务器资源。

**修复**：
```python
async def chat_with_tools_stream(self, ...):
    tasks = []
    try:
        analysis_task = asyncio.create_task(_run_need_analysis())
        tasks.append(analysis_task)
        vector_task = asyncio.create_task(self._query_context_docs_with_timeout(user_query))
        tasks.append(vector_task)
        
        # ... 正常流程 ...
        
    finally:
        for task in tasks:
            if not task.done():
                task.cancel()
                try:
                    await task
                except asyncio.CancelledError:
                    pass
```

### H5. `sqlite_product_search_service.get_products_by_ids` 连接泄漏

**文件**: `sqlite_product_search_service.py` L154

**问题**: `sqlite3.connect()` 的 `with` 语句仅管理事务，**不关闭连接**。

```python
# 当前代码（连接未关闭）
with sqlite3.connect(self.db_path) as conn:
    rows = conn.execute(sql, clean_ids).fetchall()
# conn 仍然打开
```

**修复**：
```python
conn = sqlite3.connect(self.db_path)
try:
    conn.row_factory = sqlite3.Row
    rows = conn.execute(sql, clean_ids).fetchall()
finally:
    conn.close()
```

## 七、架构层面建议

### 7.1 引入 Pipeline 模式

当前流程是线性的大方法调用链，建议改为 Pipeline：

```python
class ChatPipeline:
    stages = [
        NeedAnalysisStage(),
        VectorSearchStage(),
        RAGRankStage(),
        ToolLoopStage(),
        FinalReplyStage(),
    ]
    
    async def execute(self, ctx: ChatContext) -> AsyncGenerator[SSEEvent, None]:
        for stage in self.stages:
            async for event in stage.run(ctx):
                yield event
```

### 7.2 统一存储层

当前存在三种存储：
- JSON 文件（对话历史）→ 并发不安全
- SQLite（商品数据）→ 只读
- ChromaDB（向量库）→ 只读

建议将对话历史也迁入 SQLite，统一管理。

### 7.3 配置与代码分离

将业务规则（品类过滤、同义词映射等）从代码中抽离，集中管理：
- `attribute_ontology.json` 已有良好基础
- 扩展品类约束配置
- 查询模板配置化

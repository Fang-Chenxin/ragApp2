# 1 分钟答辩稿

老师好，我们这个项目做的是一个电商导购型原生 Android 对话应用，核心目标是把用户的模糊购物需求，变成可检索、可核验、可展示的推荐结果。

后端采用 FastAPI + Chroma + SQLite 的组合：先做向量召回，再做 RAG 核验和 SearchPlan 规划，最后只从数据库核验过的商品白名单里生成推荐，尽量避免大模型编造商品信息。推荐结果还会回查商品价格和默认 SKU，保证输出尽量可信。

客户端是 Kotlin 原生 Android，不是 Web 套壳。它通过 SSE 接收后端流式事件，边生成边渲染思考过程、正文和商品卡片，商品卡片还可以直接点击跳转落地页。

整体来说，这个项目已经完成了“模糊需求理解、库内商品检索、推荐理由生成、原生流式展示”的完整闭环。当前相对不足的是实时库存治理和图片、语音等多模态采集，这两项可以作为后续扩展方向。
